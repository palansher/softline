Гипервизор - это процесс, который помогает разделить операционную систему и запущенные приложения
от аппаратной части.

Докер - это сервис для запуска приложений в контейнерах

Контейнер - это элемент Docker, который содержит среду для выполнения Вашей программы
Среда - это набор программ, которые исползьует Ваша программа

Docker Engine - это ПО, которое позволяет создавать, разворачивать и администрировать контейнерные приложения

Docker Engine состоит из компонентов:
1) Docker Daemon - это сервер, который управляет объектами Docker: образами, контейнерами, сетями,
томами
2) REST API - это интерфейс, с помощью которого программы взаимодействуют между собой
3) Docker CLI - это клиентская утилита командной строки через которую пользователи взаимодействуют с Докер

Основные ключевые понятия в Докер:
1) Образ (Image) - это шаблон с инструкциями для создания контейнера
2) Dockerfile - это текстовый файл с инструкциями для сборки образа
3) Контейнер - это запущенный образ
4) Registry(Реестр) - место хранения образов
5) Volume(Том) - постоянное хранилище данных вне контейнера

Основные команды в Docker
1) Вывести список запущенных контейнеров: docker ps
2) Вывести все существующие контейнеры: docker ps -a
3) Вывести все локальные образы: docker images
4) Удаление контейнера: docker rm имя_контейнера
5) Загрузка образа без запуска контейнера: docker pull имя_образа
6) Запуск контейнера на основе образа. Если локального образа нет, то он скачается с docker hub
   и будет запущен. Команда называется docker run имя_образа
7) Если контейнер запущен, то удалить Вы его не сможете до остановки контейнера командой
docker stop id_контейнера
8) Для удаления образа из Докер используем команду docker rmi id_образа
9) docker pull python:3.14-slim
10) docker-compose up -d : команда запуска файла docker-compose
Тип файла yml. Если файл называется не docker-compose, то запускать такой файл нужно
командой: docker-compose -f имя_файла up

Оркестрация контейнеров - это автоматизация управления, развертывания групп контейнеров (например, Докер)

Оркестратор решает основные задачи:
1) Развертывание - запуск контейнеров на нужных серверах
2) Отказоустойчивость - перезапуск упавших контейнеров
