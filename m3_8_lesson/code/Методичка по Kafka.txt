Базовые термины:

1) Apache Kafka - это распределенная платформа для потоковой передачи событий
2) Kadka Cluster - это группа из нескольких брокеров Kafka, работающих совместно для обеспечения
масштабируемости и отказоустойчивости

Брокер 1 - контроллер, т.е. специальный брокер, управляющий кластером. Он координирует выборы
лидеров партиций. Лидер партиции - это главная партиция в которую приходят данные от Producer и
на основе этих данные, данные тиражируются на реплики партиций

Преимущества кластера:
1) Отказоустойчивость - при падении одного брокера другие продолжают работать
2) Масштабируемость - можно добавить брокеры для увеличения производительсности
3) Балансировка нагрузки - данные распределяются между брокерами


3) Структура события (сообщения)
  - Ключ - необязательное поле, используеся для контроля порядка сообщения. Если ключ сообщений
  одинаковый, то все эти сообщения с общим ключом отправляются в общую партицию.

  - Значение(Value) - JSON объект

  - Заголовки (Headers) - опциональные метанные в формате ключ-значение

  - Отметка времени (Timestamp) - время создания события

4) Топик - это имя потока в котором публикуются сообщения.

5) Партиции. Топики делятся на партиции. Каждая партиция упорядоченная. Сообщения в одной партиции
всегда упорядочены. Цель партиции - распределение нагрузки топика не несколько севреров (брокеров)

6) Смещение (Offset) - это последовательный номер сообщения внутри партиции

7) Реплкикация - это механизм отказоустойчивости. Каждая партиция копируется на несколько брокеров Кафка
Leader реплика - это основная реплика, все операции чтения и записи идут через нее
Follower реплика - точная копия данных из leader реплики

8) Брокер - это один сервер в кластере Кафка. Кафка состоит из множества брокеров.

Каждая партиция топика потребляется только одним консьюмером из группы. Координацию между брокерами
распределяет Kakfa Raft.

9) Connector - это механизм, обеспечивающий взаимодействие с внешними системами
SourceConnector - импортирует данные из внешней системы, например, из postgres
SinkConnector - экспортирует данные из топика Кафка во внешнюю систему

10) KSQL - это механизм для потоковых запросов. Кафка может выполнять запросы к СУБД без написания кода на
каком-либо языке.

Существует 3 гарантии доставки сообщений

1) At-least-once (минимум 1 раз отпарвляем). Сообщение гарантированно будет доставлено, но может быть
обработано повторно. Допускается дублирование сообщений. Если дубликаты не нужны при отправке,
то используем параметр enable.idempotence: True

2) At-most-once (Максимум 1 раз) - сообщение может быть потеряно, но повторно обработано не будет.
Продюсер отправляет сообщение и не ждет подтверждения от брокера

3) Excactly-once (ровно 1 раз) - гарантирована обработа 1 раз

Итоги:
At-most-once: потери (возможны),дубликатов (нет), скорость (высокая)
At-least-once: потери (нет),дубликатов (да), скорость (среняя)
Excactly-once: потери (нет),дубликатов (нет), скорость (низкая)

Методы отправки сообщений:
1) Отправил и забыл (не работаем с ответом от брокера)
producer.send()
....

2) Синхронная отправка
metaData = producer.send(record).get
