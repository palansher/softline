Структура JWT токена
1) Header
HS256 - симметричное шифрование (используется 1 ключ)
RS256 - асимметричное шифрование

2) Payload - полезная нагрузка, которая содержит данные на основе которых м/с предоставляет доступ.
Payload состоит из claims:
- sub : назнчение токена
- aud : аудитория для которой предназначен токен
- exp : срок жизни токена
- iat : время создания токена

3) Signature - подпись, которая генерируется следующим образом:
 Закодированные в base64 заголовок и полезная нагрузка объединяются через точку. Результат данного
 хэширования это подпись
