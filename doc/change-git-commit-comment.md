# изменения комментариев для старых комитов

## Вариант 1: Если нужно исправить САМЫЙ ПОСЛЕДНИЙ коммит

Это делается буквально в две команды:

   1. Откройте терминал
   2. Введите команду для изменения текста:

   `git commit --amend -m "Новый правильный текст коммита"`

   1. Отправьте изменения на сервер в принудительном режиме:

   git push --force-with-lease

------------------------------

## Вариант 2: Если коммит старый (находится глубже в истории)

Здесь нам поможет встроенный интерактивный rebase.

   1. Посчитайте, каким по счету сверху является нужный коммит (например, 3-м).
   2. В терминале введите команду:

   git rebase -i HEAD~3

   (замените 3 на ваше число)
   3. откроется текстовый файл со списком коммитов. Перед нужным коммитом замените слово pick на reword (или просто на букву r).
   4. Сохраните файл (Ctrl + S) и закройте вкладку.
   5. Сразу после этого откроется еще одна вкладка — в ней сотрите старый текст, напишите новый, снова сохраните (Ctrl + S) и закройте вкладку.
   6. Обязательно обновите сервер, иначе он не примет изменения:

   git push --force-with-lease

## Если Git не пошел дальше сам или предложил `rebase --continue`

Перейдите в терминал и введите команду: `git rebase --continue`

Git перейдет к следующему коммиту, который вы пометили как reword, и снова откроет вкладку для редактирования.

Повторяйте эти действия (изменил -> сохранил -> закрыл вкладку -> continue), пока в терминале не появится сообщение Successfully rebased.

После этого останется только сделать `git push --force-with-lease`
