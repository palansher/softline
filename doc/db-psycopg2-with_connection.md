# Что делает with connection: в psycopg2?

В большинстве библиотек Python (например, при работе с файлами with open(...)) контекстный менеджер with нужен для того,
чтобы автоматически закрыть ресурс в конце (сделать .close()).

Но в psycopg2 логика работы with connection: совершенно другая! Он создан исключительно для управления транзакциями. По задумке авторов:

При входе в блок with транзакция продолжается.

Если код внутри блока выполнился успешно — with автоматически делает connection.commit().

Если внутри блока произошла ошибка (Exception) — with автоматически делает connection.rollback().

При этом само соединение с базой данных with никогда не закрывает.
