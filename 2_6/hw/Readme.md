Верстка простого демонстрационного многостраничного интернет-магазина автомобилей.

---

## 🛠️ Технологический стек

- **HTML5** — семантическая разметка страниц.
- **CSS3** — кастомные стили, анимация переходов, адаптивная подгонка медиаконтента.
- **Bootstrap 5.3** — компонентная база, адаптивная сетка (Grid System) и утилитарные классы.
- **Vanilla JavaScript (ES6+)** — динамическое управление DOM, асинхронные запросы (Fetch API), валидация форм.
- **JSON** — структурированное хранение данных каталога товаров.

## 🚀 Ключевые особенности и технические решения

### 1. Архитектура и Структура (MPA)

Логика разделена на три физические страницы (`index.html`, `catalog.html`, `contacts.html`).

### 2. Блочная вёрстка и адаптивность

- Вся вёрстка выполнена на базе **DIV/Блочной модели** с активным использованием **Flexbox** и **Bootstrap Grid**. (через Bootstrap)
- Интерфейс является полностью **Responsive** и адаптируется под любые разрешения экранов.
- Для прижатия подвала (footer) к низу экрана на страницах с малым количеством контента использована связка Flexbox-утилит `d-flex flex-column min-vh-100` на теге `body` и `mt-auto` на теге `footer`.

### 3. Динамический каталог товаров (Data-Driven UI)

- Данные о товарах (6 карточек автомобилей) отделены от разметки и вынесены в изолированный файл `data/cars.json`.
- Заполнение каталога происходит асинхронно с помощью **Fetch API**. Скрипт считывает JSON, динамически генерирует HTML-код карточек через шаблонные строки и безопасно встраивает их в DOM с помощью метода `insertAdjacentHTML`.
- Реализована обработка ошибок (`.catch()`), выводящая пользователю понятное уведомление в случае сбоя сети при загрузке JSON.

### 4. Оптимизация медиаконтента

- Для предотвращения «перекоса» карточек из-за разных пропорций загружаемых изображений применено CSS-свойство `object-fit: cover` внутри фиксированных контейнеров. Это гарантирует сохранение единой высоты карточек при любом разрешении.
- Внедрен механизм Fallback** для изображений: через атрибут `onerror` на тегах `img` настроена автоматическая подстановка заглушки (Placeholder), если физический файл картинки автомобиля отсутствует на сервере.

### 5. Интерактив и валидация

- **Форма обратной связи:** Реализована клиентская валидация полей (обязательность заполнения, проверка формата телефона через `type="tel"`). Перехват отправки формы (`e.preventDefault()`) имитирует асинхронную отправку данных на сервер и сбрасывает поля (`form.reset()`).
- События для кнопок «Добавить в корзину» делегируются динамически сразу после успешного рендеринга карточек из JSON.

Разрешение в пикселях (Размер)
Для карточек в каталоге: Оптимальный размер — 800 × 533 пикселей (для 3:2) или 800 × 450 пикселей (для 16:9).

Для главного баннера (на Главной странице): Там картинка крупнее, поэтому лучше взять размер побольше, например 1200 × 800 пикселей или 1920 × 1080 пикселей.
