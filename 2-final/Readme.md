Консольное приложение на Python для безопасного управления учетными записями пользователей (регистрация, смена паролей, просмотр списка логинов) с хранением данных в формате JSON.

Программа включает в себя продвинутую систему валидации паролей на основе регулярных выражений.

## Основной функционал

- Добавление пользователя: Проверка логина на уникальность и оценка пароля по 5-балльной шкале (длина, регистр, цифры, спецсимволы).

- Изменение пароля: Обновление пароля существующего пользователя с обязательной валидацией надежности.

- Вывод пользователей: Отображение списка всех зарегистрированных логинов (без раскрытия паролей).

- Безопасное завершение: Корректный выход из программы.

## Архитектура проекта и компоненты

Приложение спроектировано в соответствии с принципами ООП и разделено на независимые модули для обеспечения чистоты кода и строгого разделения обязанностей (Separation of Concerns):

```text
└── project/
    ├── main.py                  # Точка входа (главный скрипт запуска)
    ├── application.py           # Контроллер интерфейса и меню (CLI)
    ├── user_manager.py           # Управление данными и JSON-файлом
    └── password_validator.py     # Валидация надежности паролей (RegEx)
```

### main.py (Точка входа)

Роль: Инициализатор (Entry Point).

### application.py (Класс Application)

Роль: Контроллер интерфейса (CLI).

Обязанности: Отображение главного меню и обработка ввода пользователя.

Координация взаимодействия между менеджером пользователей и валидатором паролей.

Организация интерактивных циклов (например, запросы на повторный ввод пароля при слабой/средней надежности).

### user_manager.py (Класс UserManager)

Роль: Слой работы с данными (Data Access Layer).

Обязанности:

Чтение и сохранение базы данных в файл `users.json`.

Автоматическое определение пути к файлу относительно расположения самого скрипта (через pathlib).

Инкапсуляция данных: проверка существования логина (user_exists), добавление (add_user) и обновление данных без риска создания дубликатов.

### password_validator.py (Класс PasswordValidator)

Роль: Компонент бизнес-логики (Валидатор).

Обязанности:

Проверка паролей на соответствие 5 критериям безопасности с помощью регулярных выражений (re).

Подсчет баллов надежности. Использование сырых строк (r"") для безопасной изоляции спецсимволов и исключения конфликтов с интерпретатором Python.

## Технологический стек

Язык: Python 3.10+

Библиотеки: json, re, pathlib, os.

Code Style: Строгая аннотация типов (Type Hinting), документирование в стиле Google Docstrings, соответствие PEP 8.

## Техническое задание

Программа должна выводить следующее меню:

1) Добавить пользователя
2) Изменить пароль
3) Вывести пользователей
4) Выход

Если пользователь выбирает пункт 1, программа предлагает ввести идентификатор пользователя.
Она должна проверить, существует ли введенный идентификатор в списке.

Если это так, программа выводит соответствующее сообщение и предлагает выбрать другой идентификатор.

После того как пользователь введет допустимый идентификатор, программа запрашивает пароль.
Паролю начисляется по одному баллу за соответствие перечисленным ниже условиям:

- пароль должен содержать не менее 8 символов;
- пароль должен включать буквы верхнего регистра;
- пароль должен включать буквы нижнего регистра;
- пароль должен включать цифры;
- пароль должен включать хотя бы один специальный
символ: !, £, $, %, &,

Если пароль получает всего 1 или 2 балла, он должен быть отклонен с формулировкой, что он является слабым.

Если у пароля 3 или 4 балла, выводится сообщение о том, что его можно улучшить.
Спросите пользователя, хочет ли он повторить попытку.

Если пароль набрал 5 баллов, сообщите, что он является сильным.

В конец файла . json должны добавляться только допустимые логины и пароли.

Если пользователь выберет пункт 2, он должен будет ввести логин пользователя.
Проверьте, существует ли логин в списке.

Если это так, предложите пользователю изменить пароль и сохраните изменения в файле json.

Убедитесь в том, что программа только изменяет существующий пароль, а не создает новую запись. Если пользователь выберет пункт 3, выведите только логины пользователей без паролей.

Наконец, при выборе пункта 4 программа должна завершиться
